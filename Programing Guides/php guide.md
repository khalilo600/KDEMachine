# PHP Guide: Comprehensive Learning Outline

This guide provides a structured overview of PHP (Hypertext Preprocessor), a widely-used open-source general-purpose scripting language that is especially suited for web development and can be embedded into HTML. It covers fundamental concepts, control flow, OOP, web development essentials, database interaction, error handling, and best practices.

---

## I. Getting Started and Core Concepts

### A. What is PHP?

PHP is a server-side scripting language designed for web development but also used as a general-purpose programming language. It was originally created by Rasmus Lerdorf in 1994.

*   **Server-side:** PHP code is executed on the server, generating HTML that is then sent to the client's web browser.
*   **Open-source:** Freely available and maintained by a large community.
*   **Embedded in HTML:** PHP code can be directly embedded within HTML markup.
*   **Dynamic Content:** Used to create dynamic web pages, interact with databases, handle forms, and much more.

### B. Why Use PHP?

*   **Ease of Learning:** Relatively easy to learn for beginners.
*   **Wide Adoption:** Powers a significant portion of the web (WordPress, Facebook, Wikipedia).
*   **Large Ecosystem:** Extensive libraries, frameworks (Laravel, Symfony, CodeIgniter), and CMS platforms.
*   **Database Support:** Excellent support for various databases, especially MySQL.
*   **Cross-Platform:** Runs on various operating systems (Linux, Windows, macOS) and web servers (Apache, Nginx, IIS).

### C. Installation and Setup (XAMPP, MAMP, Docker)

To run PHP, you need a web server (like Apache or Nginx) and a PHP interpreter.

1.  **Local Development Environments:**
    *   **XAMPP (Cross-Platform Apache, MySQL, PHP, Perl):** A free and open-source cross-platform web server solution stack package. Easy to install on Windows, macOS, and Linux.
    *   **MAMP (macOS, Apache, MySQL, PHP):** Similar to XAMPP but specifically for macOS.
    *   **WAMP (Windows, Apache, MySQL, PHP):** Similar to XAMPP but specifically for Windows.
2.  **Docker:** For more advanced and isolated development environments, Docker is highly recommended.

    ```bash
    # Example Docker Compose for PHP, Nginx, MySQL
    # docker-compose.yml

    services:

          MYSQL_ROOT_PASSWORD: root_password
          MYSQL_DATABASE: mydatabase
        volumes:
          - db_data:/var/lib/mysql
    volumes:
      db_data:
    ```

    ```dockerfile
    # Dockerfile.php
    FROM php:8.2-fpm
    WORKDIR /var/www/html
    RUN docker-php-ext-install pdo pdo_mysql
    ```

### D. Basic Syntax (`<?php ?>`, Comments)

PHP code is enclosed within `<?php` and `?>` tags. Anything outside these tags is treated as plain HTML.

```php
<!DOCTYPE html>
<html>
<head>
    <title>PHP Example</title>
</head>
<body>
    <h1>
        <?php echo "Hello, World!"; ?>
    </h1>

    <?php
        // This is a single-line comment

        /*
         * This is a multi-line comment
         * that can span across several lines.
         */

        # This is another single-line comment (shell-style)

        echo "<p>This is a paragraph generated by PHP.</p>";
    ?>
</body>
</html>
```

### E. Variables and Data Types

Variables in PHP start with a `$` sign. PHP is a loosely typed language, meaning you don't have to declare the data type of a variable.

1.  **Scalar Types:**
    *   **String:** Sequence of characters (e.g., `"Hello"`, `'World'`).
    *   **Integer:** Whole numbers (e.g., `10`, `-5`).
    *   **Float (or Double):** Numbers with a decimal point (e.g., `3.14`, `0.5`).
    *   **Boolean:** `true` or `false`.

2.  **Compound Types:**
    *   **Array:** Stores multiple values in a single variable.
    *   **Object:** Instances of classes.

3.  **Special Types:**
    *   **NULL:** A variable with no value assigned.
    *   **Resource:** A special variable, holding a reference to an external resource (e.g., a database connection).

    ```php
    <?php
    $name = "Alice";        // String
    $age = 30;              // Integer
    $price = 19.99;         // Float
    $is_admin = true;       // Boolean
    $colors = ["red", "green", "blue"]; // Array (indexed)
    $user = [               // Array (associative)
        "name" => "Bob",
        "age" => 25
    ];
    $null_var = null;       // NULL

    echo "Name: " . $name . "<br>";
    echo "Age: " . $age . "<br>";
    echo "First color: " . $colors[0] . "<br>";
    echo "User name: " . $user["name"] . "<br>";
    ?>
    ```

### F. Operators

*   **Arithmetic Operators:** `+`, `-`, `*`, `/`, `%` (modulus), `**` (exponentiation).
*   **Assignment Operators:** `=`, `+=`, `-=`, `*=`, `/=`, `%=`, `.=` (string concatenation assignment).
*   **Comparison Operators:** `==` (equal), `===` (identical - value and type), `!=` (not equal), `!==` (not identical), `<`, `>`, `<=`, `>=`.
*   **Logical Operators:** `and` (`&&`), `or` (`||`), `xor`, `!` (not).
*   **Increment/Decrement Operators:** `++$var`, `$var++`, `--$var`, `$var--`.
*   **String Operator:** `.` (concatenation).

    ```php
    <?php
    $x = 10;
    $y = 3;
    echo $x + $y; // 13
    echo $x % $y; // 1

    $message = "Hello";
    $message .= " World!"; // $message is now "Hello World!"
    echo $message;

    $a = 5;
    $b = "5";
    var_dump($a == $b);  // true (loose comparison)
    var_dump($a === $b); // false (strict comparison)
    ?>
    ```

### G. Outputting Data (`echo`, `print`)

*   **`echo`:** Can output one or more strings. It's slightly faster than `print`.
*   **`print`:** Can only output a single string and returns 1 (always successful).

    ```php
    <?php
    echo "Hello, PHP!";
    echo "This is ", "multiple ", "strings.";
    print "This is a single string.";

    $name = "John";
    echo "My name is $name."; // Variables are parsed inside double quotes
    echo 'My name is $name.'; // Variables are NOT parsed inside single quotes
    ?>
    ```

## II. Control Flow and Functions

### A. Conditional Statements

1.  **`if`, `else if`, `else`:** Executes different blocks of code based on conditions.

    ```php
    <?php
    $temperature = 25;
    if ($temperature > 30) {
        echo "It's hot!";
    } elseif ($temperature > 20) {
        echo "It's warm.";
    } else {
        echo "It's cold.";
    }
    ?>
    ```

2.  **`switch`:** Executes code based on matching `case` values.

    ```php
    <?php
    $day = "Monday";
    switch ($day) {
        case "Monday":
            echo "Start of the week.";
            break;
        case "Friday":
            echo "End of the week!";
            break;
        default:
            echo "Mid-week.";
    }
    ?>
    ```

### B. Looping Constructs

1.  **`for` loop:** Repeats a block of code a specified number of times.

    ```php
    <?php
    for ($i = 0; $i < 5; $i++) {
        echo $i . "<br>"; // 0, 1, 2, 3, 4
    }
    ?>
    ```

2.  **`while` loop:** Repeats a block of code as long as a condition is true.

    ```php
    <?php
    $count = 0;
    while ($count < 3) {
        echo $count . "<br>"; // 0, 1, 2
        $count++;
    }
    ?>
    ```

3.  **`do...while` loop:** Similar to `while`, but the block of code is executed at least once.

    ```php
    <?php
    $i = 0;
    do {
        echo $i . "<br>"; // 0
        $i++;
    } while ($i < 0);
    ?>
    ```

4.  **`foreach` loop:** Iterates over elements in an array or objects.

    ```php
    <?php
    $colors = ["red", "green", "blue"];
    foreach ($colors as $color) {
        echo $color . "<br>";
    }

    $user = ["name" => "Jane", "age" => 30];
    foreach ($user as $key => $value) {
        echo $key . ": " . $value . "<br>";
    }
    ?>
    ```

### C. Functions

Functions are blocks of code designed to perform a particular task.

1.  **Defining and Calling Functions:**

    ```php
    <?php
    function greet($name) {
        return "Hello, " . $name . "!";
    }

    echo greet("Alice"); // "Hello, Alice!"
    ?>
    ```

2.  **Parameters and Return Values:**
    *   Functions can accept parameters (arguments).
    *   Functions can return a value using the `return` statement.

3.  **Variable Scope (Local, Global, Static):
    *   **Local:** Variables declared inside a function are local to that function.
    *   **Global:** Variables declared outside functions are global. To use them inside a function, use the `global` keyword or the `$GLOBALS` superglobal array.
    *   **Static:** A static variable retains its value between function calls.

    ```php
    <?php
    $global_var = "I'm global";

    function testScope() {
        $local_var = "I'm local";
        echo $local_var . "<br>";

        global $global_var; // Access global variable
        echo $global_var . "<br>";

        static $static_count = 0;
        $static_count++;
        echo "Static count: " . $static_count . "<br>";
    }
    testScope(); // Static count: 1
    testScope(); // Static count: 2
    ?>
    ```

4.  **Anonymous Functions and Closures:** Functions without a specified name, often used as callbacks. Closures can inherit variables from the parent scope using the `use` keyword.

    ```php
    <?php
    $greeting = "Hello";
    $sayHello = function($name) use ($greeting) {
        echo "$greeting, $name!<br>";
    };
    $sayHello("Bob"); // "Hello, Bob!"
    ?>
    ```

5.  **Arrow Functions (PHP 7.4+):** A more concise syntax for anonymous functions with a single expression.

    ```php
    <?php
    $factor = 2;
    $numbers = [1, 2, 3];
    $doubled = array_map(fn($n) => $n * $factor, $numbers);
    print_r($doubled); // Array ( [0] => 2 [1] => 4 [2] => 6 )
    ?>
    ```

## III. Arrays and Strings

### A. Indexed Arrays

Arrays with numeric keys, starting from 0.

```php
<?php
$fruits = ["Apple", "Banana", "Cherry"];
echo $fruits[0]; // Apple
$fruits[] = "Date"; // Add to end
print_r($fruits);
?>
```

### B. Associative Arrays

Arrays with named keys.

```php
<?php
$person = [
    "name" => "Jane Doe",
    "age" => 28,
    "city" => "New York"
];
echo $person["name"]; // Jane Doe
$person["email"] = "jane@example.com";
print_r($person);
?>
```

### C. Multidimensional Arrays

Arrays containing other arrays.

```php
<?php
$students = [
    ["name" => "Alice", "grade" => "A"],
    ["name" => "Bob", "grade" => "B"]
];
echo $students[0]["name"]; // Alice
?>
```

### D. Array Functions

PHP has a rich set of built-in array functions.

*   `count($array)`: Returns the number of elements.
*   `sort($array)`: Sorts an indexed array in ascending order.
*   `rsort($array)`: Sorts an indexed array in descending order.
*   `asort($array)`: Sorts an associative array by value.
*   `ksort($array)`: Sorts an associative array by key.
*   `array_push($array, $value1, ...)`: Pushes one or more elements onto the end of an array.
*   `array_pop($array)`: Pops the last element off the end of array.
*   `array_merge($array1, $array2, ...)`: Merges one or more arrays.
*   `in_array($needle, $haystack)`: Checks if a value exists in an array.
*   `array_keys($array)`: Returns all the keys or a subset of the keys of an array.
*   `array_values($array)`: Returns all the values of an array.

    ```php
    <?php
    $numbers = [4, 2, 8, 1];
    sort($numbers);
    print_r($numbers); // Array ( [0] => 1 [1] => 2 [2] => 4 [3] => 8 )

    $merged = array_merge([1, 2], [3, 4]);
    print_r($merged); // Array ( [0] => 1 [1] => 2 [2] => 3 [3] => 4 )
    ?>
    ```

### E. String Functions

PHP provides many functions for manipulating strings.

*   `strlen($string)`: Returns the length of a string.
*   `str_word_count($string)`: Counts the number of words in a string.
*   `str_replace($search, $replace, $subject)`: Replaces all occurrences of a substring.
*   `substr($string, $start, $length)`: Returns a part of a string.
*   `trim($string)`: Removes whitespace or other characters from the beginning and end of a string.
*   `strpos($haystack, $needle)`: Finds the position of the first occurrence of a substring.
*   `strtolower($string)`, `strtoupper($string)`: Converts string to lowercase/uppercase.

    ```php
    <?php
    $text = "  Hello World!  ";
    echo strlen($text) . "<br>"; // 16 (includes spaces)
    echo trim($text) . "<br>"; // "Hello World!"
    echo str_replace("World", "PHP", $text) . "<br>"; // "  Hello PHP!  "
    ?>
    ```

## IV. Object-Oriented Programming (OOP)

PHP has full support for object-oriented programming.

### A. Classes and Objects

*   **Class:** A blueprint for creating objects.
*   **Object:** An instance of a class.

    ```php
    <?php
    class Car {
        // Properties
        public $brand;
        public $color;

        // Methods
        public function start() {
            return "The " . $this->color . " " . $this->brand . " is starting.";
        }
    }

    // Create an object (instance of the Car class)
    $myCar = new Car();
    $myCar->brand = "Toyota";
    $myCar->color = "blue";

    echo $myCar->start(); // The blue Toyota is starting.
    ?>
    ```

### B. Properties and Methods

*   **Properties:** Variables within a class.
*   **Methods:** Functions within a class.
*   **`$this` keyword:** Refers to the current object.

### C. Constructors and Destructors

*   **Constructor (`__construct()`):** A special method called automatically when an object is created.
*   **Destructor (`__destruct()`):** A special method called automatically when an object is destroyed or the script ends.

    ```php
    <?php
    class Person {
        public $name;

        public function __construct($name) {
            $this->name = $name;
            echo "Person {$this->name} created.<br>";
        }

        public function __destruct() {
            echo "Person {$this->name} destroyed.<br>";
        }
    }

    $p1 = new Person("Alice"); // Person Alice created.
    // Script ends, Person Alice destroyed.
    ?>
    ```

### D. Access Modifiers (Public, Protected, Private)

Control the visibility of properties and methods.

*   **`public`:** Accessible from anywhere.
*   **`protected`:** Accessible within the class itself and by inheriting classes.
*   **`private`:** Accessible only within the class that defines it.

### E. Inheritance

A class can inherit properties and methods from another class. The child class extends the parent class.

```php
<?php
class Animal {
    protected $name;

    public function __construct($name) {
        $this->name = $name;
    }

    public function eat() {
        return $this->name . " is eating.";
    }
}

class Dog extends Animal {
    public function bark() {
        return $this->name . " says Woof!";
    }
}

$dog = new Dog("Buddy");
echo $dog->eat() . "<br>";  // Buddy is eating.
echo $dog->bark() . "<br>"; // Buddy says Woof!
?>
```

### F. Interfaces

Define a contract that classes must adhere to. A class implementing an interface must implement all methods declared in the interface.

```php
<?php
interface Shape {
    public function getArea();
}

class Circle implements Shape {
    private $radius;

    public function __construct($radius) {
        $this->radius = $radius;
    }

    public function getArea() {
        return M_PI * $this->radius * $this->radius;
    }
}
?>
```

### G. Abstract Classes

Classes that cannot be instantiated directly and must be inherited by other classes. They can have abstract methods (without implementation) that child classes must implement.

### H. Traits

Used to reuse methods in independent classes that belong to different class hierarchies.

### I. Namespaces

Solve the problem of name collisions between classes, functions, and constants when combining code from different libraries.

```php
<?php
namespace App\Models;

class User {
    // ...
}

namespace App\Controllers;

use App\Models\User; // Import the User class from App\Models

class UserController {
    public function showUser() {
        $user = new User();
        // ...
    }
}
?>
```

## V. Web Development Fundamentals

### A. Superglobals

PHP provides several predefined global arrays (superglobals) that are always accessible, regardless of scope.

*   **`$_GET`:** An associative array of variables passed to the current script via the URL parameters.
*   **`$_POST`:** An associative array of variables passed to the current script via the HTTP POST method.
*   **`$_REQUEST`:** An associative array that by default contains the contents of `$_GET`, `$_POST`, and `$_COOKIE`.
*   **`$_SESSION`:** An associative array containing session variables available to the current script.
*   **`$_COOKIE`:** An associative array of variables passed to the current script via HTTP Cookies.
*   **`$_SERVER`:** An associative array containing information like headers, paths, and script locations.
*   **`$_FILES`:** An associative array of items uploaded to the current script via the HTTP POST method.

### B. Handling Forms (GET vs. POST)

*   **GET:** Data is appended to the URL. Suitable for non-sensitive data and when users might want to bookmark the page. Limited data size.
*   **POST:** Data is sent in the HTTP request body. Suitable for sensitive data (passwords) and large amounts of data.

```html
<!-- index.html -->
<form action="process.php" method="POST">
    <label for="name">Name:</label>
    <input type="text" id="name" name="username"><br><br>
    <label for="email">Email:</label>
    <input type="email" id="email" name="useremail"><br><br>
    <input type="submit" value="Submit">
</form>
```

```php
<?php
// process.php
if ($_SERVER["REQUEST_METHOD"] == "POST") {
    $name = htmlspecialchars($_POST["username"]); // Sanitize input
    $email = htmlspecialchars($_POST["useremail"]);
    echo "Hello, " . $name . "! Your email is " . $email . ".";
}
?>
```

### C. Sessions and Cookies

*   **Sessions:** Store user data on the server for the duration of their visit.
    *   `session_start()`: Must be called at the beginning of every script that uses sessions.
    *   `$_SESSION`: Used to store and retrieve session variables.
*   **Cookies:** Store user data on the client's browser.
    *   `setcookie()`: Used to set a cookie.
    *   `$_COOKIE`: Used to retrieve cookie values.

    ```php
    <?php
    session_start(); // Start the session

    // Set session variable
    $_SESSION["favcolor"] = "green";
    echo "Favorite color is " . $_SESSION["favcolor"] . ".<br>";

    // Set a cookie (expires in 1 hour)
    setcookie("username", "Alice", time() + 3600, "/");
    if (isset($_COOKIE["username"])) {
        echo "Welcome back, " . $_COOKIE["username"] . ".<br>";
    }
    ?>
    ```

### D. File Uploads

Handled using the `$_FILES` superglobal.

```html
<!-- form.html -->
<form action="upload.php" method="POST" enctype="multipart/form-data">
    Select image to upload:
    <input type="file" name="fileToUpload" id="fileToUpload">
    <input type="submit" value="Upload Image" name="submit">
</form>
```

```php
<?php
// upload.php
if (isset($_POST["submit"])) {
    $target_dir = "uploads/";
    $target_file = $target_dir . basename($_FILES["fileToUpload"]["name"]);
    $uploadOk = 1;
    $imageFileType = strtolower(pathinfo($target_file, PATHINFO_EXTENSION));

    // Check if image file is a actual image or fake image
    $check = getimagesize($_FILES["fileToUpload"]["tmp_name"]);
    if ($check !== false) {
        echo "File is an image - " . $check["mime"] . ".";
        $uploadOk = 1;
    } else {
        echo "File is not an image.";
        $uploadOk = 0;
    }

    // Check if $uploadOk is set to 0 by an error
    if ($uploadOk == 0) {
        echo "Sorry, your file was not uploaded.";
    } else {
        if (move_uploaded_file($_FILES["fileToUpload"]["tmp_name"], $target_file)) {
            echo "The file ". htmlspecialchars( basename( $_FILES["fileToUpload"]["name"])). " has been uploaded.";
        } else {
            echo "Sorry, there was an error uploading your file.";
        }
    }
}
?>
```

### E. Redirects (`header()`)

Used to redirect the user to a different page. Must be called before any output is sent to the browser.

```php
<?php
header("Location: /dashboard.php");
exit(); // Always call exit() after a redirect
```

## VI. Database Interaction (MySQL with PDO)

PHP's PDO (PHP Data Objects) extension provides a lightweight, consistent interface for accessing databases.

### A. Connecting to a Database

```php
<?php
$host = 'localhost';
$db   = 'mydatabase';
$user = 'root';
$pass = 'password';
$charset = 'utf8mb4';

$dsn = "mysql:host=$host;dbname=$db;charset=$charset";
$options = [
    PDO::ATTR_ERRMODE            => PDO::ERRMODE_EXCEPTION,
    PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC,
    PDO::ATTR_EMULATE_PREPARES   => false,
];

try {
    $pdo = new PDO($dsn, $user, $pass, $options);
    echo "Connected to database successfully!<br>";
} catch (\PDOException $e) {
    throw new \PDOException($e->getMessage(), (int)$e->getCode());
}
?>
```

### B. Performing CRUD Operations (Create, Read, Update, Delete)

1.  **Create (Insert):

    ```php
    <?php
    // Assuming $pdo is connected
    $stmt = $pdo->prepare("INSERT INTO users (name, email) VALUES (?, ?)");
    $stmt->execute(["John Doe", "john@example.com"]);
    echo "New record created successfully.<br>";
    ?>
    ```

2.  **Read (Select):

    ```php
    <?php
    $stmt = $pdo->query("SELECT id, name, email FROM users");
    while ($row = $stmt->fetch()) {
        echo "ID: " . $row['id'] . ", Name: " . $row['name'] . ", Email: " . $row['email'] . "<br>";
    }
    ?>
    ```

3.  **Update:

    ```php
    <?php
    $stmt = $pdo->prepare("UPDATE users SET email = ? WHERE name = ?");
    $stmt->execute(["john.doe@example.com", "John Doe"]);
    echo "Record updated successfully.<br>";
    ?>
    ```

4.  **Delete:

    ```php
    <?php
    $stmt = $pdo->prepare("DELETE FROM users WHERE name = ?");
    $stmt->execute(["John Doe"]);
    echo "Record deleted successfully.<br>";
    ?>
    ```

### C. Prepared Statements (Preventing SQL Injection)

Prepared statements are crucial for security. They separate the SQL query from the data, preventing SQL injection attacks.

```php
<?php
$name = "Jane Doe";
$email = "jane@example.com";

$stmt = $pdo->prepare("INSERT INTO users (name, email) VALUES (:name, :email)");
$stmt->bindParam(':name', $name);
$stmt->bindParam(':email', $email);
$stmt->execute();
echo "New record created using prepared statement.<br>";
?>
```

### D. Fetching Data

*   `fetch()`: Fetches the next row from a result set.
*   `fetchAll()`: Returns an array containing all of the remaining rows in the result set.

## VII. Error Handling and Debugging

### A. Error Reporting (`error_reporting`, `ini_set`)

Control which PHP errors are reported.

```php
<?php
// Turn on all error reporting
error_reporting(E_ALL);
ini_set('display_errors', 1); // Display errors on screen (for development)
ini_set('log_errors', 1);     // Log errors to a file (for production)
ini_set('error_log', '/path/to/php-error.log');
?>
```

### B. Custom Error Handlers

You can define your own function to handle PHP errors.

```php
<?php
function myErrorHandler($errno, $errstr, $errfile, $errline) {
    echo "<b>Custom Error:</b> [$errno] $errstr<br>";
    echo "Error on line $errline in $errfile<br>";
    // Don't execute PHP's internal error handler
    return true;
}

set_error_handler("myErrorHandler");

// Trigger an error
echo $undefined_variable;
?>
```

### C. Exceptions (`try...catch...finally`)

PHP's exception handling allows for a more structured way to manage errors.

```php
<?php
function divide($numerator, $denominator) {
    if ($denominator == 0) {
        throw new Exception("Division by zero.");
    }
    return $numerator / $denominator;
}

try {
    echo divide(10, 2) . "<br>";
    echo divide(5, 0) . "<br>"; // This will throw an exception
    echo "This line will not be executed.<br>";
} catch (Exception $e) {
    echo "Caught exception: " . $e->getMessage() . "<br>";
} finally {
    echo "Finally block executed.<br>";
}
?>
```

### D. Debugging Tools (Xdebug)

Xdebug is a powerful PHP extension for debugging, profiling, and code coverage analysis. It integrates with IDEs like VS Code.

## VIII. Advanced Topics and Best Practices

### A. Composer (Dependency Management)

Composer is a dependency manager for PHP. It allows you to declare the libraries your project depends on and it will install and manage them for you.


### B. PSR Standards (Coding Style, Autoloading)

PSR (PHP Standard Recommendation) are PHP standards recommendations published by the PHP Framework Interop Group (PHP-FIG). They provide guidelines for coding style, autoloading, interfaces, etc., promoting interoperability.

*   **PSR-1:** Basic Coding Standard
*   **PSR-2:** Coding Style Guide (superseded by PSR-12)
*   **PSR-4:** Autoloader (for classes)

### C. Autoloading Classes

Composer automatically generates an autoloader (`vendor/autoload.php`) that loads classes as they are needed, so you don't have to manually `require` each class file.

```php
<?php
require 'vendor/autoload.php'; // Include Composer's autoloader

use Monolog\Logger;
use Monolog\Handler\StreamHandler;

// Create a logger
$log = new Logger('name');
$log->pushHandler(new StreamHandler('app.log', Logger::WARNING));

// Add records to the log
$log->warning('Foo');
$log->error('Bar');
?>
```

### F. Security Best Practices (Input Validation, Output Escaping, Password Hashing)

*   **Input Validation:** Always validate and sanitize all user input.
    *   Use `filter_var()` or `preg_match()`.
    *   Never trust user input.
*   **Output Escaping:** Escape all output to prevent XSS (Cross-Site Scripting) attacks.
    *   Use `htmlspecialchars()` or template engine's escaping features.
*   **Password Hashing:** Never store plain text passwords. Use strong hashing algorithms.
    *   `password_hash()`: To hash a password.
    *   `password_verify()`: To verify a password against a hash.

    ```php
    <?php
    $password = "mysecretpassword";
    $hashed_password = password_hash($password, PASSWORD_DEFAULT);
    echo "Hashed password: " . $hashed_password . "<br>";

    if (password_verify($password, $hashed_password)) {
        echo "Password is valid!<br>";
    } else {
        echo "Invalid password.<br>";
    }
    ?>
    ```
